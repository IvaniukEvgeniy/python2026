# Практическое задание 35.2

## Задания

### Описание задачи
В предыдущем блоке вы научились объявлять функции, передавать аргументы и возвращать результат. Теперь вы переходите к управлению аргументами: использованию значений по умолчанию и правильному порядку их передачи. Это поможет избежать частых ошибок, особенно при работе с изменяемыми типами данных.

Ваша задача — написать функцию для управления списком задач (to-do list). Функция должна безопасно добавлять задачи в список, учитывая возможность передачи дополнительных параметров, и корректно работать с изменяемыми значениями по умолчанию.

### Пример входных и выходных данных

**Пример 1: Базовое добавление задачи**
```python
my_tasks = []
add_task(my_tasks, "Купить молоко")
print(my_tasks)
# Ожидаемый вывод: [{'name': 'Купить молоко', 'priority': 'medium', 'tags': []}]
```

**Пример 2: Добавление задачи с указанием приоритета и тегов**
```python
my_tasks = []
add_task(my_tasks, "Позвонить маме", priority="high", tags=["семья", "важно"])
add_task(my_tasks, "Прочитать книгу", tags=["развитие"])
print(my_tasks)
# Ожидаемый вывод:
# [
#   {'name': 'Позвонить маме', 'priority': 'high', 'tags': ['семья', 'важно']},
#   {'name': 'Прочитать книгу', 'priority': 'medium', 'tags': ['развитие']}
# ]
```

**Пример 3: Проверка на ловушку изменяемого значения по умолчанию**
```python
list1 = []
add_task(list1, "Задача 1")
add_task(list1, "Задача 2")
print(list1)  # Должно быть две задачи в list1

list2 = []
add_task(list2, "Задача A")
print(list2)  # Должна быть только "Задача A", а не ["Задача 1", "Задача 2", "Задача A"]
```

### Критерии проверки и ограничения
1.  Функция `add_task` должна принимать следующие параметры в строгом порядке:
    *   `task_list` (обязательный): список, в который добавляется задача.
    *   `task_name` (обязательный): строка с названием задачи.
    *   `priority` (необязательный): строка, обозначающая приоритет. По умолчанию должно быть значение `"medium"`.
    *   `tags` (необязательный): список тегов для задачи. По умолчанию должен создаваться новый пустой список при каждом вызове, если аргумент не передан (используйте `None`-паттерн).
2.  Функция формирует словарь с ключами `"name"`, `"priority"` и `"tags"` и добавляет его в переданный список `task_list`. Функция ничего не возвращает (`None`).
3.  При вызове можно использовать как позиционные, так и именованные аргументы, соблюдая правильный порядок: сначала обязательные позиционные (`task_list`, `task_name`), затем необязательные именованные (`priority`, `tags`).
4.  Запрещено использовать `*args`, `**kwargs`, распаковку, а также темы областей видимости и глобальных переменных.

### Решение задачи (псевдокод)
1.  Объяви функцию `add_task` с четырьмя параметрами: `task_list`, `task_name`, `priority`, `tags`.
2.  Для параметра `priority` установи значение по умолчанию `"medium"`.
3.  Для параметра `tags` установи значение по умолчанию `None`.
4.  Внутри функции:
    a.  Проверь, равен ли параметр `tags` значению `None`.
    b.  Если да, создай новый пустой список и присвой его переменной, которая будет использоваться для тегов.
    c.  Если нет, используй переданный список `tags`.
5.  Создай словарь с ключами:
    *   `"name"` — значение из `task_name`.
    *   `"priority"` — значение из `priority`.
    *   `"tags"` — список тегов (полученный на шаге 4).
6.  Добавь этот словарь в конец списка `task_list`.
7.  Заверши выполнение функции (не возвращай явно никакого значения).

### Рекомендации
-   Вынести в отдельную функцию: В данном случае логика добавления задачи является цельной и простой, поэтому выносить часть кода в отдельную функцию не требуется. Основная цель — отработать объявление функции с параметрами по умолчанию.
-   PEP8:
    -   Используйте `snake_case` для имен функций и переменных (например, `add_task`, `task_list`).
    -   После запятой, разделяющей параметры, ставьте пробел: `def add_task(task_list, task_name, priority="medium", tags=None):`.
