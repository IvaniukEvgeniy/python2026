# Практическое задание 35.1

Это задание относится к первому блоку темы "Функции". Вы уже изучили основы работы с коллекциями (списки, словари, множества) и алгоритмами. Сейчас вы переходите к ключевому инструменту структурирования кода — функциям. В этом блоке вы научитесь объявлять функции, передавать в них данные через параметры и получать результат с помощью оператора `return`. Это фундамент для всех последующих тем, связанных с функциями.

## Задания

### Описание задачи
Вы работаете над простым модулем для расчёта стоимости доставки. Ваша задача — создать функцию `calculate_delivery`, которая будет принимать два аргумента: расстояние до пункта назначения в километрах (`distance_km`) и вес посылки в килограммах (`weight_kg`). Функция должна вычислять стоимость доставки по следующему правилу:
- Базовый тариф: 300 рублей за заказ.
- За каждый полный километр расстояния добавляется 20 рублей.
- За каждый полный килограмм веса добавляется 50 рублей.

Функция должна **возвращать** итоговую сумму (целое число). Не используйте `print` внутри функции для вывода результата — только `return`.

### Пример входных и выходных данных
**Пример 1:**
*   **Входные данные для вызова функции:** `calculate_delivery(10, 2)`
*   **Что происходит:** Расстояние = 10 км, Вес = 2 кг.
*   **Расчёт:** Базовый тариф (300) + (10 км * 20) + (2 кг * 50) = 300 + 200 + 100 = 600.
*   **Ожидаемый результат (возвращаемое значение):** `600`

**Пример 2:**
*   **Входные данные для вызова функции:** `calculate_delivery(5, 0)`
*   **Что происходит:** Расстояние = 5 км, Вес = 0 кг (лёгкое письмо).
*   **Расчёт:** 300 + (5 * 20) + (0 * 50) = 300 + 100 + 0 = 400.
*   **Ожидаемый результат (возвращаемое значение):** `400`

### Критерии проверки и ограничения
1.  Функция должна быть объявлена с помощью `def` и иметь имя `calculate_delivery`.
2.  Функция должна принимать ровно два **позиционных аргумента** (параметра) в указанном порядке: сначала расстояние, затем вес.
3.  Внутри функции должен использоваться оператор `return` для возврата целочисленного результата.
4.  Не используйте значения параметров по умолчанию, именованные аргументы при вызове, `*args`, `**kwargs` или глобальные переменные. Сосредоточьтесь на базовом синтаксисе.
5.  Код вне функции (например, вызовы для проверки) может содержать `print`, но сама функция — нет.

### Решение задачи (псевдокод)
1.  Объяви функцию с именем `calculate_delivery`, которая принимает два параметра: `distance_km` и `weight_kg`.
2.  Внутри функции создай переменную для хранения базового тарифа и присвой ей значение 300.
3.  Вычисли доплату за расстояние: умножь значение параметра `distance_km` на 20.
4.  Вычисли доплату за вес: умножь значение параметра `weight_kg` на 50.
5.  Вычисли итоговую сумму: сложи базовый тариф, доплату за расстояние и доплату за вес.
6.  Верни полученную сумму как результат работы функции.

### Рекомендации
-   **Вынести в отдельную функцию:** Всё решение задачи по своей сути является одной функцией. Дополнительные функции создавать не требуется.
-   **PEP8:**
    -   Используйте осмысленные имена параметров (`distance_km`, `weight_kg`), которые сразу дают понять, что они означают и в каких единицах измерения.
    -   После объявления функции `def calculate_delivery(...):` оставьте две пустые строки перед следующим блоком кода (например, перед вызовами функции для проверки).
