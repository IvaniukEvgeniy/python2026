# Домашнее задание. Урок 33. Генераторы коллекций. list, dict, set comprehensions

## Краткая сводка

- Старайся оформлять решение лаконично, без вспомогательных циклов. Все входные данные можно брать из примеров или придумать собственные, но формат вывода должен совпадать с описанием.

## Задача 1. Повторение — «температура по дням»
- **Блок:** list comprehension.
- **Описание:** дан список температур `temperatures = [-8, -2, 5, 11, 0, 3, -6]`. Сформируй список строк `"День X: ±T°C"` только для дней с температурой > -5. Нумерация начинается с 1.
- **Критерии:** один comprehension, знак `+` добавляется явно для неотрицательных значений, исходный список не меняется.

## Задача 2. Закрепление — «список закупок»
- **Блок:** list comprehension + форматирование.
- **Описание:** получен список `orders = [("Игорь", 52000, "Казань"), ...]`. Нужно оставить покупки из городов, начинающихся на `К`, и получить строки `"Имя купил(а) на SUM"`, где суммы ≥ 50000 преобразуются в `SUMK`, остальные — `SUM₽`. Для имён на `а/я` используй «купила».
- **Критерии:** один comprehension, фильтр по городу, условное окончание суммы и глагола реализовано в выражении, исходные данные не модифицируются.

## Задача 3. Интеграция — «режим работы сотрудников»
- **Блок:** dict comprehension.
- **Описание:** список словарей `employees = [{"name": "Антон", "hours": 18, "remote": True}, ...]`. Построй словарь `work_mode`, где ключ — имя, значение — строка `"удалённо"` или `"в офисе"`, если сотрудник отработал ≥ 20 часов.
- **Критерии:** один dict comprehension, фильтр (`hours >= 20`) в конце, тернарный оператор для выбора текста, исходный список не меняется.

## Задача 4. Подготовка — «лог активности»
- **Блок:** set comprehension.
- **Описание:** строки формата `"user:action"` собраны в `logs`. Нужно получить множество пользователей, у которых действие `"login"` или `"update"`. Пропускай записи с другим форматом.
- **Пример:** `["anna:login", "bob:logout", "anna:update"]` → `{"anna"}`.
- **Критерии:** один set comprehension, корректное разделение строки, исходные данные не меняются.
