# Задание 29.4. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Вложенные структуры. Учимся читать и изменять списки словарей, чтобы доставать нужные данные из расписаний и таблиц.

## Условие задачи
Ты ведешь расписание мастерских в молодежном центре:
1. Создай список словарей с полями `day`, `time`, `topic` и `mentor`.
2. С помощью цикла распечатай расписание в формате `День: тема (ментор)`.
3. Сформируй список дней, где ментор совпадает с именем, которое задается отдельной переменной.
4. Построй новый список только с вечерними занятиями. фильтруй по времени `>= "18:00"`.
5. Подсчитай количество уникальных тем, пройдясь по вложенным элементам.

## Пример входных данных
```python
workshops = [
    {"day": "Пн", "time": "17:00", "topic": "робототехника", "mentor": "Алина"},
    {"day": "Ср", "time": "19:30", "topic": "3D печать", "mentor": "Сергей"},
    {"day": "Сб", "time": "12:00", "topic": "анимация", "mentor": "Алина"}
]
mentor_filter = "Алина"
```

## Пример выходных данных
```
Пн: робототехника (Алина)
Ср: 3D печать (Сергей)
Сб: анимация (Алина)
Дни Алины: ['Пн', 'Сб']
Вечерние занятия: [{'day': 'Ср', 'time': '19:30', 'topic': '3D печать', 'mentor': 'Сергей'}]
Уникальных тем: 3
```

## Критерии успешного выполнения
- Список состоит из словарей с одинаковыми ключами
- Вывод расписания проходит через цикл по вложенной структуре
- Фильтрация по ментору и времени оформлена отдельными списками
- Подсчет уникальных тем выполняется без хардкода значений

## Подсказки
- Используй `for entry in workshops` для чтения словарей
- Внутри цикла обращайся к полям через `entry["topic"]`
- Для выбора вечерних занятий сравнивай строки времени лексикографически
- Уникальные темы удобно собирать в `set`

## Частые ошибки и ограничения
- Не путай ключи словаря. лучше вынести строки в константы
- Не забывай очищать вспомогательные списки перед повторным использованием
- Избегай копирования всего списка ради одной фильтрации
- Не считай уникальные темы вручную. используй структуры данных
