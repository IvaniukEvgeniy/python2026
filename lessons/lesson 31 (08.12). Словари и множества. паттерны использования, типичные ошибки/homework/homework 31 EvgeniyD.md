## Задача 1. Справочник промокодов

Интернет‑магазин хранит промокоды и размеры скидок.

- **Блок:** словарь как структура быстрого доступа.
- **Дано:** список пар `codes` и список `to_check` с кодами, которые ввели пользователи.

Пример входных данных:
```python
codes = [
    ("HELLO", 5),
    ("SALE10", 10),
    ("VIP", 20),
]

to_check = ["HELLO", "UNKNOWN"]
```

### Требуется
1. Собрать словарь `discounts`, где ключ. строка промокода, значение. размер скидки.
2. Для каждого кода из `to_check` вывести строку `<код> -> <скидка>%`.
3. Если кода нет в словаре, выводить `<код> -> нет такой акции`.
4. Использовать только `dict.get(code, default)`.

### Критерии
- Правильно построен словарь из списка пар.
- Для всех проверок используется `get` с запасным значением.
- Нет падений при запросе несуществующих кодов.

## Задача 2. Учёт заказов курьеров

Сервис доставки считает, сколько заказов выполнил каждый курьер за день.

- **Блок:** мутация словарей, аккуратное удаление.

Пример входных данных:
```python
shifts = [
    ("Ivanov", 3),
    ("Petrov", 2),
    ("Ivanov", 4),
    ("Sidorov", 1),
]
finished = ["Petrov", "Unknown"]
```

### Требуется
1. Построить словарь `orders`, где ключ. фамилия курьера, значение. общее число заказов.
2. Безопасно удалить из `orders` всех курьеров из списка `finished` с помощью `pop(name, 0)`.
3. Вывести итоговый словарь в формате `Фамилия -> количество заказов`.

### Критерии
- Суммирование значений реализовано через `get` или аналогичную конструкцию.
- Для удаления всегда используется `pop` с запасным значением.
- Код не падает, если в `finished` есть фамилии, которых нет в словаре.

## Задача 3. Множества участников митапа

У тебя есть списки посещения очного и онлайн‑формата митапа.

- **Блок:** множества и операции над ними.

Пример входных данных:
```python
offline = {"Alex", "Maria", "Ivan"}
online = {"Maria", "Olga", "Sergey"}
```

### Требуется
1. Найти множество людей, которые были и офлайн, и онлайн.
2. Найти множество людей, которые были только онлайн.
3. Распечатать результат в виде:
   ```text
   Оба формата: {...}
   Только онлайн: {...}
   ```

### Критерии
- Используются операции множеств (`&`, `-`).
- Лишние преобразования в списки не используются без необходимости.
- Подписи в выводе соответствуют смыслу.

## Задача 4. Подготовка к comprehensions

Нужно подготовить данные о покупках так, чтобы их можно было легко фильтровать и преобразовывать, в том числе через генераторы коллекций.

Пример входных данных:
```python
purchases = [
    ("pizza", 850),
    ("coffee", 180),
    ("tea", 120),
    ("burger", 430),
]
threshold = 200
```

### Требуется
1. Собрать словарь `prices`, где ключ. название товара, значение. цена.
2. На основе словаря создать **новый** словарь `expensive`, куда попадут только товары с ценой не меньше `threshold`.
3. Вывести оба словаря в удобном виде.

### Критерии
- Код разделён на два понятных шага: накопление → фильтрация.
- Структура решения легко переписывается в comprehension.
- Формат вывода понятный, без лишней информации.
